# New project workflow

1. **Questions** — Ask until the system understands the idea (goals, constraints, tech preferences, edge cases).
2. **Research** — Optional: spawn parallel agents to investigate the domain.
3. **Requirements** — Extract what is v1, v2, and out of scope.
4. **Roadmap** — Create phases mapped to requirements.

**Creates:** PROJECT.md, REQUIREMENTS.md, ROADMAP.md, STATE.md, research/
